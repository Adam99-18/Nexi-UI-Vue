> 生成日期：2026-05-03  
> 包名：`nexi-ui` (当前版本 1.0.2，建议发布 1.1.0)  
> 修复范围：浏览器 F12 控制台错误/警告消除 & 组件功能可用性保障 & 组件增强

## 零、根因分析：页面白屏

### 问题现象
引入 `nexi-ui` 后页面完全空白，控制台大量组件解析失败警告：
> Failed to resolve component: a-button / a-modal / a-input / a-switch ...

### 根因
`nexi-ui` 全部组件内部依赖 `ant-design-vue` 的 `<a-*>` 组件（如 `<a-button>`、`<a-modal>` 等），但使用者 `main.js` 中仅注册了 `NexiUI`，**未注册 `Antd`**，导致所有 `<a-*>` 组件无法解析。

### 修复方案
使用者必须在入口文件中同时注册 `ant-design-vue`：

```js
// main.js — 正确用法
import { createApp } from 'vue'
import Antd from 'ant-design-vue'          // ← 必须
import 'ant-design-vue/dist/reset.css'      // ← 必须（cssinjs 需要）
import NexiUI from 'nexi-ui'
import 'nexi-ui/style.css'
import App from './App.vue'

const app = createApp(App)
app.use(Antd)    // ← 必须
app.use(NexiUI)
app.mount('#app')
```

> **注意：** ant-design-vue 4.x 使用 cssinjs 方案注入组件样式，`reset.css` 提供基础 CSS reset，组件样式在运行时自动注入。这是 peerDependencies 设计——nexi-ui 不会内联 ant-design-vue。

### 2. `antd/input.vue` — v-model 绑定只读 computed（P0 致命）

**问题：** `valuePro = computed(() => props.value)` 无 setter，`v-model:value="valuePro"` 触发：
> [Vue warn]: Write operation failed: computed value is readonly

**修复：**
```diff
- const valuePro = computed(() => props.value);
+ const valuePro = computed({
+   get: () => props.value,
+   set: (val) => emits('update:value', val),
+ });
```

**修复：**
```diff
- const currentKey = computed(() => props.value);
+ const currentKey = computed({
+   get: () => props.value,
+   set: (val) => {
+     emits("update:value", val + "");
+     emits("change", val + "");
+   },
+ });
```

### 4. `antd/switch.vue` — 缺少 v-model 事件（P0 致命）

```diff
  };
```

### 5. `antd/form.vue` — reactive 对象被重新赋值破坏响应式（P0 致命）

**问题：** `formState = reactive(props.value)` 后 watch 中直接 `formState = val` 把 Proxy 换成普通对象。

**修复：**
```diff
- let formState = reactive(props.value);
+ const formState = reactive({ ...props.value });

  watch(() => props.value, (val: any) => {
-   formState = val;
+   Object.keys(formState).forEach((key) => delete formState[key]);
+   Object.assign(formState, val);
- });
+ }, { deep: true });

  // reset 同步修复
- formState = resetObj(cloneDeep(props.value));
+ Object.keys(formState).forEach((key) => delete formState[key]);
+ Object.assign(formState, resetObj(cloneDeep(props.value)));
```

同步移除 3 处 `console.log`。

### 12. `antd/modal-tip.vue` & `antd/modal.vue` — getContainer 返回 null 导致 modal 无法渲染（P0 致命）

**问题：** `getContainer` 查询 `document.querySelector(props.layout)`（默认 `.layout-wrap`），使用者 DOM 中不存在该选择器时返回 `null`，ant-design-vue 的 `<a-modal>` 收起无效容器导致 modal 无法弹出。

**修复：**
```diff
  const getContainer = () => {
    if (typeof window !== "undefined") {
-     return document.querySelector(props.layout) as HTMLElement;
+     return document.querySelector(props.layout) || document.body;
    }
+   return document.body;
  };
```
`.layout-wrap` 是 nexi-ui 配套 layout 组件的容器选择器。当使用者未使用 nexi-ui layout 时，自动降级到 `document.body`。

### 14. `antd/search-form.vue` — 响应式网格布局增强（P3 增强）

**问题：** 原布局使用 `flex-wrap` + 固定 `200px` 宽度，在大屏上每行仅 1 个表单项，空间利用率低。无 `a-row`/`a-col` 响应式栅格能力。表单项 `border` 为内联 style，外部无法覆盖。

**修复：**

- 使用 `<a-row :gutter="props.gutter">` + `<a-col>` 替换 `flex-wrap` div
- 每列默认响应式断点：`xs:24 / sm:24 / md:12 / lg:12 / xl:8 / xxl:6`（大屏 4 列）
- 每个 column 可通过 `colProps` 覆盖各断点 span
- 新增 `gutter` / `wrapperClass` / `wrapperStyle` props
- 表单项 border 移入 scoped CSS 类 `.search-form-item`，外部可通过 `:deep()` 覆盖

```diff
+ props 新增：
+   gutter?: number | [number, number];     // 默认 [10, 16]
+   wrapperClass?: string;                   // 默认 ''
+   wrapperStyle?: Record<string, any>;      // 默认 {}
+
  // template：
- <div class="flex-wrap">
+ <a-row :gutter="props.gutter">
+   <a-col v-bind="{ xs:..., sm:..., md:..., lg:..., xl:..., xxl:... }">
```

### 18. `antd/switch.vue` — 点击事件与 a-switch 内部交互冲突（P0 致命）

**问题：** 外层 `<div @click="switchStatus">` 包裹 `<a-switch>`，`<a-switch>` 自身有内部点击处理。点击时事件可能被 `<a-switch>` 消费导致外层 click 不触发，或双重触发导致状态错乱。在 NexiTable 的 `bodyCell` 插槽中使用时尤甚。

**修复：**
```diff
- <div class="flex-h" @click="switchStatus">
-   <a-switch :disabled="disabled" :checked="props.checked"></a-switch>
-   <a v-if="props.checked">启用</a>
-   <span v-else>禁用</span>
- </div>
+ <div class="flex-h">
+   <a-switch :disabled="disabled" :checked="props.checked" @change="switchStatus" />
+   <a v-if="props.checked" @click="switchStatus(false)">启用</a>
+   <span v-else @click="switchStatus(true)">禁用</span>
+ </div>
```
```diff
- const switchStatus = () => {
-   emits("update:checked", !props.checked);
-   emits("change", !props.checked);
- };
+ const switchStatus = (val: boolean) => {
+   if (props.disabled) return;
+   emits("update:checked", val);
+   emits("change", val);
+ };
```

- `<a-switch>` 的 `@change` 事件直接驱动状态更新，避免事件冲突
- 文本标签点击也显式传递目标值
- 新增 `disabled` 守卫

## 五、使用者接入指南

### 最小可用配置

```js
// main.js
import { createApp } from 'vue'
import Antd from 'ant-design-vue'
import 'ant-design-vue/dist/reset.css'
import NexiUI from 'nexi-ui'
import 'nexi-ui/style.css'
import App from './App.vue'

const app = createApp(App)
app.use(Antd)     // ← 必须先注册 ant-design-vue
app.use(NexiUI)
app.mount('#app')
```

### 组件用法示例

```vue
<template>
  <!-- 按钮 -->
  <nexi-button type="primary" @click="handleClick">提交</nexi-button>

  <!-- 输入框：v-model:value -->
  <nexi-input v-model:value="keyword" placeholder="搜索" />

  <!-- 开关：v-model:checked -->
  <nexi-switch v-model:checked="enabled" />

  <!-- 标签页：v-model:value -->
  <nexi-tabs v-model:value="tab" :list="[
    { label: 'Tab1', value: '1' },
    { label: 'Tab2', value: '2' }
  ]" />

  <!-- 弹窗提示：v-model:visible -->
  <nexi-modal-tip
    v-model:visible="showTip"
    title="提示"
    content="确认执行？"
    @ok="handleOk"
  />

  <!-- 搜索表单（V1.1 响应式） -->
  <nexi-search-form
    :columns="[
      { field: 'name', label: '名称', component: 'NexiInput',
        componentProps: { placeholder: '请输入' } },
      { field: 'status', label: '状态', component: 'NexiSelect',
        componentProps: { options: [{ value: '1', label: '启用' }] } },
      { field: 'date', label: '日期', component: 'RangePicker',
        colProps: { xl: { span: 12 }, xxl: { span: 8 } } }
    ]"
    :gutter="[10, 16]"
    search-text="搜索"
    reset-text="重置"
    wrapper-class="my-search"
    @search="handleSearch"
    @reset="handleReset"
  />
</template>
```
